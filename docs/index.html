🔒 End-to-end encryption

AES-256-GCM with scrypt key derivation. Your password never leaves your machine. Telegram only sees ciphertext.

⚡ Parallel transfers

Upload 3 chunks, download 5 chunks simultaneously. 100 MB default chunk size.

🔄 Resumable uploads

CRC32-protected progress files survive interruptions. Resume with --resume.

🛡️ Data safety

Atomic config writes, sequential index access, cached lookups, crash-safe upload/delete/stream. Your data comes first.

💾 Git-like backups

Incremental snapshots with retention policies. Restore to any point in time.

📁 FUSE mount

Mount your vault as a local filesystem. On-demand chunk streaming with LRU cache.

🌐 WebDAV server

Access files over HTTP from any device. macOS Finder, Windows Explorer, mobile apps.

🖥️ Terminal UI (beta)

Interactive file browser with detail panel. Still improving — CLI commands are the most reliable.

🔀 Pipeable I/O

cat file | tvt push - · tvt cat file | jq · tvt ls --json

⏰ Auto-backup

Schedule backups with systemd timers or cron. Watch directories for live changes.

Quick start

# 1) Get API credentials at https://my.telegram.org
export TELEGRAM_API_ID=your_id
export TELEGRAM_API_HASH=your_hash

# 2) Login
tvt login

# 3) Setup (interactive, validates channel, sends test message)
tvt setup

# 4) Go!
tvt push photo.jpg
tvt ls
tvt pull photo.jpg

Your password never leaves your machine. Each chunk gets a random 16-byte salt and 12-byte nonce. BLAKE3 hashes verify integrity at every stage. If you lose your password, there is no recovery.

Project Structure

  • cli.py — Click CLI, command dispatch, friendly errors
  • core.py — TeleVault class: upload, download, stream
  • telegram.py — MTProto client, channel ops, index
  • models.py — FileMetadata, ChunkInfo, VaultIndex
  • chunker.py — File splitting, ChunkWriter, BLAKE3
  • crypto.py — AES-256-GCM, scrypt, streaming
  • compress.py — zstd compression
  • config.py — Config dataclass, atomic persistence
  • fuse.py — FUSE driver with on-demand streaming
  • preview.py — Terminal preview from headers
  • tui.py — Textual TUI with detail panel

Key Concepts

  • All data in a private Telegram channel as pinned messages + reply chains
  • Files chunked → hashed → compressed → encrypted → uploaded
  • VaultIndex maps file IDs to message IDs (version-gated, 3 retries)
  • Cached index message IDs for O(1) lookups, asyncio.Lock for concurrency safety
  • All CLI errors handled by run_async() — friendly messages only
  • Entry points: tvt and televault
  • 168 tests in tests/
